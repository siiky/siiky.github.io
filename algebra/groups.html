This post is about groups (whouldathunkit).

1 Conventions/Notation

∀v:P\forall v: P means PP is true for all possible vv
∃v:P\exists v: P means PP is true for at least one vv
∃!v:P\exists! v: P means PP is true for exactly one vv
∀v1:∀v2:P\forall v1: \forall v2: P will be abbreviated as ∀v1,v2:P\forall v1, v2: P

Sets are written as its elements surrounded by brackets. For example, ∅\emptyset is the empty set and {a,b,c}\{ a, b, c \} is a set with the elements aa, bb and cc. Sets comprehension will be written as {expr:vars,restrictions}\{ expr : vars, restrictions \} and the cardinal of a set will be written as #S\#S.

ℕ\mathbb{N} is the set of natural numbers (no zero).
ℕ0\mathbb{N}_0 is the set of natural numbers (and zero).
ℤ\mathbb{Z} is the set of integers.

The modulo operator will be recurrent throughout this post. Some examples: 4mod5≡9mod5≡44\ \bmod 5 \equiv 9\ \bmod 5 \equiv 4, 5mod2≡15\ \bmod 2 \equiv 1. This operator is called % in C, modulo in Scheme and mod in Haskell. Another useful definition, is: p≡q(modn)⇔pmodn≡qmodnp \equiv q\ (\bmod n) \Leftrightarrow p\ \bmod n \equiv q\ \bmod n
|| will be used to mean divides, as in a|ba | b means a divides b, b is a multiple of a, or, with modulo, a|b⇔b≡0(moda)a | b \Leftrightarrow b \equiv 0 (\bmod a)

We will be needing the concept of a Congruential Equivalence Class later on. They are written as [m]n[m]_n (∀n∈ℕ,m∈ℤ\forall n \in \mathbb{N}, m \in \mathbb{Z}) and are defined as [m]n={x∈ℤ:x≡m(modn)}[m]_n = \{ x \in \mathbb{Z} : x \equiv m\ (\bmod n) \}.

Depending on context, ++ and ×\times will either be the usual addition and multiplication of numbers, or addition and multiplication of classes.
Addition and multiplication of classes are defined as [p]n+[q]n=[p+q]n[p]_n + [q]_n = [p+q]_n and [p]n×[q]n=[p×q]n[p]_n \times [q]_n = [p \times q]_n. Some examples: 2+2=42+2=4, [2]3+[1]3=[2+1]3=[0]3[2]_3 + [1]_3 = [2+1]_3 = [0]_3

2 What is a Group?!

2.1 In general

A group is just a pair (S,O)(S, O), where SS is a set and OO is a binary operation on SS (i.e., O:S×S→SO \colon S \times S \to S) with the following properties:

  1. Associativity
    ∀a,b,c∈S:O(a,O(b,c))=O(O(a,b),c)\forall a, b, c \in S: O(a, O(b, c)) = O(O(a, b), c)
  2. Identity
    ∃!id∈S:∀a∈S:O(a,id)=O(id,a)=a\exists! id \in S: \forall a \in S: O(a, id) = O(id, a) = a
  3. Inverse
    ∀a∈S:∃!a′∈S:O(a,a′)=O(a′,a)=id\forall a \in S: \exists! a' \in S: O(a, a') = O(a', a) = id
    id always has inverse, itself

There is one extra property:

  1. Commutativity
    ∀a,b∈S:O(a,b)=O(b,a)\forall a, b \in S: O(a, b) = O(b, a)

A group with a commutative operation is called a commutative group or an abelian group.

Given G=(G,O)G = (G, O) a group, GG will be used to refer both to the group itself and its associated set.

A group GG is said to be infinite if #G\#G is infinite, and finite if #G\#G is finite.

That is the generic definition. We will focus on groups with integer sets and ++ or ×\times as the operation, however, so to make things easier:

2.2 Integers

When the operation is ++ we call the identity element null element and represent it with 0.
When the operation is ×\times we call the identity element unity element and represent it with 1.

When the operation is ++ we call the inverse of an element aa its symmetric and represent it as −a\minus a.
When the operation is ×\times we call the inverse of an element aa its inverse and represent it as a−1a^{-1}.

2.2.1 (ℕ0\mathbb{N}_0, +)

ℕ0\mathbb{N}_0 is the set of natural numbers (with zero), and ++ is the usual addition on natural numbers.
Is it a group?

  1. ∀a,b,c∈ℕ0:a+(b+c)=(a+b)+c\forall a, b, c \in \mathbb{N}_0: a + (b + c) = (a + b) + c
  2. 0 is the identity of the group, because ∀a∈ℕ0:a+0=0+a=a\forall a \in \mathbb{N}_0: a + 0 = 0 + a = a
  3. ∀a,b∈ℕ:a+b≠0\forall a, b \in \mathbb{N}: a + b \neq 0 (Note that ℕ=ℕ0\{0}\mathbb{N} = \mathbb{N}_0 \setminus \{0\})

ℕ0\mathbb{N}_0 together with ++ does not satisfy property 3, so it is not a group.

2.2.2 (ℤ\mathbb{Z}, +)

ℤ\mathbb{Z} is the set of integers, and ++ is the usual addition on integers. Is it a group?

  1. a, b, c in Z: a + (b + c) = (a + b) + c
  2. 0 is the identity of the group, because a in Z: a + 0 = 0 + a = a
  3. a in Z: ! a’ in Z: a + a’ = 0

ℤ\mathbb{Z} satisfies all 3 requirements so it is a group. We also know that addition is commutative, so ℤ\mathbb{Z} is an abelian group (see property 4).

2.2.3 Other groups and non-groups

  • (R, +), where R is the set of real numbers, is a group
  • (R\{0}, *) is also a group
  • (R, *) is not a group, because 0 has no inverse (R3)
  • (N, +) is not a group, because there is no identity and no inverse (R2 and R3)
  • (N, *) is not a group, because there is no inverse (R3)
  • (Z\{0}, *) is not a group, because other than 1 and -1, no element has inverse (R3)

3 What’s next?

Some useful definitions follow.

3.1 Subgroup

Given a group G, H is called a subgroup of G if H is contained in G and it is also a group, and we write H <= G. Examples:

  • \forall (G, +) group: (G, +) <= (G, +)
  • Trivial Subgroup: \forall (G, +) group: ({0}, +) <= (G, +)
  • (2Z, +) <= (Z, +)

A subgroup H of a group G that is not G itself (H != G, or H is strictly contained in G) is called a Proper Subgroup, and we write H < G. Examples:

  • Trivial Subgroup: \forall (G, +) group: ({0}, +) < (G, +)
  • (2Z, +) < (Z, +)

3.2 Multiples/Powers of an element

Given an element a of an additive group, a + ... + a (n times) can be written as n * a. Special case: 0 * a = 0. Given an element a of a multiplicative group, a * ... * a (n times) can be written as a ^ n. Special case: a ^ 0 = 1.

For negative n, n * a = -((-n) * a) = (-n) * (-a). For negative n, a ^ n = (a ^ (-n)) ^ -1 = (a ^ -1) ^ (-n).

3.3 Order of an element

The order of an element a is the minimum number of times it must be operated with itself until it reaches the identity, and is written as o(a). If, no matter how many times you operate the element, it doesn’t reach the identity, its order is said to be infinite.

A more rigorous definition is:

  • a has infinite order if n in N: n * a != 0
  • a has finite order k, i.e., o(a) = k if:
    1. k is in N
    2. k * a = 0
    3. n in N: n * a = 0 => n <= k

In particular, the order of the identity is 1, and the identity is the only element with order 1. Useful fact: G group: a in G: o(a) | #G. From this comes the fact that, in a finite group, no element has infinite order.

3.4 Generated Subgroup

Given an additive group (G, +) and an element a, <a> = { n * a : n in Z } is a subgroup of G and is called the Subgroup of G generated by a. In particular, if G = <a>, a is said to generate G, or that G is generated by a. Note that #<a> = o(a). An example of this is <1> = <-1> = Z.

4 ℤn\mathbb{Z}_n Groups

You can group integers together according to their (mod n), make a set out of them, and define a group with it. These groups are called Zn, for some n in N, and are defined as Zn = { [m]n : m in { 0, …, n-1 } }.

These will get repetitive after Z2, but the reason why they’re here will become clear.

4.1 (ℤ1\mathbb{Z}_1, +)

According to definition above, Z1 = { [0]1 }, but you can go further here:

[0]1 = (def [m]n) { x : x in Z, x = 0 (mod 1) } = (def (mod n)) { x : x in Z, x % 1 = 0 % 1 } = (0 % 1 = 0) { x : x in Z, x % 1 = 0 } = (every integer is a multiple of 1) { x : x in Z } = (def Z) Z

Z1 is a trivial group (#Z1 = 1), so it isn’t that interesting, other than Z being its only element.

Some things we can find about it:

  • Since it is a trivial group, its only subgroup is itself
  • [0]1 is the identity, so o([0]1) = 1
  • Z1 = <[0]1>

4.2 (ℤ2\mathbb{Z}_2, +)

When n = 2 we get Z2 = { [0]2, [1]2 }.

[0]2 = (def [m]n) { x : x in Z, x = 0 (mod 2) } = (def (mod n)) { x : x in Z, x % 2 = 0 % 2 } = (0 % 2 = 0) { x : x in Z, x % 2 = 0 } = (x % 2 = 0 => k in Z: x = 2 * k) { 2 * x : x in Z } = (def 2Z) 2Z

So [0]2 = 2Z is the set of even integers. You can probably guess by now, but:

[1]2 = (def [m]n) { x : x in Z, x = 1 (mod 2) } = (def (mod n)) { x : x in Z, x % 2 = 1 % 2 } = (1 % 2 = 1) { x : x in Z, x % 2 = 1 } = (x % 2 = 1 => k in Z: x = 2 * k + 1) { 2 * x + 1 : x in Z } = (def 2Z+1) 2Z+1

And with that you see that [1]2 is the set of odd integers.

Some things we can find about it:

  • [0]2 is the identity, so o([0]2) = 1
  • [1]2 != [0]2, but [1]2 + [1]2 = [1 + 1]2 = [2]2 = [0]2, so o([1]2) = 2
  • <[0]2> = { [0]2 } < Z2
  • <[1]2> = Z2

4.3 (ℤ3\mathbb{Z}_3, +)

Z3 = { [0]3, [1]3, [2]3 }

I’ll skip showing how to get to the definition of each of the classes from now. Also, you may have already noticed, but if not, [0]n is the set of multiples of n, nZ.

[0]3 = 3Z [1]3 = 3Z + 1 [2]3 = 3Z + 2

And now things about Z3:

  • o([0]3) = 1
  • o([1]3) = 3; you may also have noticed that o([1]n) = n. This has to do with the fact that n in Z: n * 1 = n. So n * [1]n = [n * 1]n = [n]n = [0]n
  • o([2]3) = 3; [2]3 != [0]3, 2 * [2]3 = [1]3 != [0]3, 3 * [2]3 = [0]3 This one has to do with the fact that 2 and 3 are coprime, which means lcm(2, 3) = 6, and 6 / 2 = 3
  • <[0]3> = { [0]3 } < Z3
  • <[1]3> = <[2]3> = Z3

4.4 (ℤ4\mathbb{Z}_4, +)

Z4 = { [0]4, [1]4, [2]4, [3]4 }

[0]4 = 4Z [1]4 = 4Z + 1 [2]4 = 4Z + 2 [3]4 = 4Z + 3

Once again, things about this group:

  • o([0]4) = 1
  • o([2]4) = 2
  • o([1]4) = o([3]4) = 4
  • <[2]4> = { [0]4, [2]4 } < Z4
  • <[1]4> = <[3]4> = Z4

Now something interesting happened here! Remember that G group: a in G: o(a) | #G Because of this we know that the only possible orders are 1, 2 and 4. This also means that it is possible for a subgroup of cardinal 2 to exist. In this case, the only one is <[2]4>. Why, though, did this happen with Z4, but not with Z1, Z2 or Z3? Z1 is obvious: #Z1 = 1, so the only possible subgroup is itself. Z2 is also easy: its only elements are [0]2 and [1]2, and we know that n in N: <[0]n> = { [0]n } and that n in N: <[1]n> = Zn For Z3 it’s not as clear. The hint is, again: G group: a in G: o(a) | #G Both 2 and 3 are prime, which means, their only divisors are 1 and themselves. So it’s not possible for a subgroup of Z3 with cardinal 2 to exist.

4.5 (ℤ5\mathbb{Z}_5, +)

Try this one

Summing up: n in N: Zn has more than one proper subgroup <=> n is not prime or n in N: Zn has exactly one proper subgroup <=> n is prime

Proving this is easy: Let n be in N.

Suppose that Zn has more than one proper subgroup. We want to prove that n is not prime. There is a proper subgroup H such that #H > 1. Since H is a proper subgroup, then #H < n. From this, and the fact that #H | n, we can conclude n is not prime.

Now the other way: Suppose n is not prime. We want to prove that Zn has more than one proper subgroup. Since n is not prime, then k in N: 1 < k < n and k | n Let k be such a number. Then a in Zn: o(a) = k Let a be such an element. o(a) = k => #<a> = k We know that <a> <= Zn and that k < n, so <a> < Zn.
