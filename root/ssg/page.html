EDIT: WIP.

Simple set and forget static site generator

  • Markdown -> HTML (keep overall look of markdown)
  • Filesystem structure traversing (tree like)

Markdown -> HTML

  • Headers
  • Code (blocks and inline)
    • Blocks
    • Inline
  • URLs (inline and by reference)
    • Inline
    • Reference
  • Images (inline and by reference)
    • Inline
    • Reference

Headers

From:

# H1
## H2
### H3
#### H4
##### H5
###### H6

to:

<H1># H1</H1>
<H2>## H2</H2>
<H3>### H3</H3>
<H4>#### H4</H4>
<H5>##### H5</H5>
<H6>###### H6</H6>

Code

Blocks

From:

```some\_lang\_here
some code here

to:

```html
```some\_lang\_here<CODE>
some code here
</CODE>```

Inline

From:

`some code here`

to:

`<CODE>some code here</CODE>`

URLs

Inline

From:

[some text](some_url)

to:

<A HREF="some_url">some text</A>

Reference

From:

[some text][ref]
[ref]: some_url

to:

<A HREF="some_url">some text</A>

Images

Inline

From:

![some text](some_url)

to:

<IMG SRC="some_url" ALT="some text">

Reference

From:

![some text][ref]
[ref]: some_url

to:

<IMG SRC="some_url" ALT="some text">

Filesystem Structure Traversing

From:

.
├── words
│  ├── slpod.md
│  └── ssg.md
└── index.md

to:

.
├── words
│  ├── slpod.md
│  ├── slpod.html
│  ├── ssg.md
│  └── ssg.html
├── index.md
└── index.html

Inside index.html:

.
└── words
├── <A href="words/slpod.html">slpod.md</A>
└── <A href="words/ssg.html">ssg.md</A>
